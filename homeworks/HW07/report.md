## 2. Protocol

Опишите ваш "честный" unsupervised-протокол.

- Препроцессинг: 
  - **Масштабирование**: StandardScaler применён ко всем признакам каждого датасета. Это критически важно для distance-based методов (KMeans, DBSCAN, Agglomerative), так как без масштабирования признаки в больших шкалах доминируют.
  - **Обработка пропусков**: не требуется (все датасеты не содержат пропусков)
  - **Кодирование категориальных признаков**: не требуется (все датасеты содержат только числовые признаки)
  - Препроцессинг оформлен через Pipeline для каждого датасета отдельно, чтобы обеспечить консистентность применения к разным моделям.

- Поиск гиперпараметров:
  - **KMeans**: подбор k в диапазоне [2, 20]. Для каждого k фиксировались `random_state=42` и `n_init=10` для воспроизводимости. Критерий выбора лучшего k - максимизация silhouette score.
  - **DBSCAN**: подбор `eps` в диапазоне [0.1, 2.0] с шагом 0.1 при фиксированном `min_samples=5`. Пропускались конфигурации с более чем 50% шума или менее 2 кластеров. Критерий - максимизация silhouette score на non-noise точках.
  - Критерий выбора лучшей модели для каждого датасета: максимизация silhouette score (сравнение KMeans и DBSCAN).

- Метрики: 
  - **silhouette_score**: выше - лучше, диапазон [-1, 1]. Оценивает насколько хорошо объекты разделены на кластеры.
  - **davies_bouldin_score**: ниже - лучше, минимум 0. Оценивает среднее отношение внутрикластерного расстояния к межкластерному.
  - **calinski_harabasz_score**: выше - лучше. Оценивает отношение межкластерной дисперсии к внутрикластерной.
  - Для DBSCAN метрики считались только на non-noise точках (label != -1), так как метрики не определены для шума.

- Визуализация: 
  - **PCA(2D)**: применён с `random_state=42` для каждого датасета. Визуализированы лучшие решения (лучшая модель по silhouette score). PCA показывает глобальную структуру данных в первых двух главных компонентах.
  - t-SNE: не использовался (опционально).

## 6. Conclusion

4-8 коротких тезисов: чему научились про кластеризацию, метрики и корректный протокол unsupervised-эксперимента.

1. Масштабирование признаков критически важно для distance-based методов кластеризации. Без масштабирования признаки в больших шкалах доминируют, что искажает результаты.

5. Проверка устойчивости (например, через ARI при разных random_state) помогает оценить надёжность результатов, особенно для KMeans.

6. PCA(2D) визуализация полезна для понимания глобальной структуры данных, но не всегда отражает реальные кластеры в многомерном пространстве.
